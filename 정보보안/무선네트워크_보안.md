# 무선네트워크 보안

1. 무선 랜 네트워크
- 무선 네트워크 접속
- 무선 랜 기술 발전 동향
  - 무선 랜의 특징
    - 유선 랜에 비해 통신의 한계가 불분명하며, 거리, 접속 방향 등에 대한 정보를 얻기 위해 별도 프로그램 필요
    - 네트워크가 노출되는 무선 랜의 취약한 보안 때문에 무선 랜을 제거하는 경우도 발생
    - 무선 랜은 2.4Ghz 통신망을 이용하며, 표준규약으로 802.11을 사용하고 802.11a ~ 802.11n 등으로 발전
    - 초기 802.11b(11Mbps) >> 802.11g(52Mbps), 802.11n(100Mbps)로 발전

|년도|프로토콜 이름| 특징 |
|:---|:-----------:|:----:|
|1997|802.11	   |최초의 무선 프로토콜|
|1999|802.11a      |5Ghz/2Mbps(WIFI-5) 무선 전화기, 블루투스 / 더 넓은 전파 대역을 사용할 수 있다. 전파 투과성과 회절성이 떨어져 통신 단절 현상이 심하며 802.11b와 비호환|
|1999|802.11b      |2.4Ghz/11Mbps WEP 방식의 보안 구현|
|2003|802.11g      |2.4Ghz/54Mbps                     |
|2004|802.11i      |2.4Ghz/54Mpbs 802.11b 표준 WPA 채택으로 보안성을 강화|
|2009|802.11n      |2.4Ghz/5Ghz 다중 입력 다중 출력 기술과 대역폭 손실 최소화|

- 무선 랜 장,단점
  - 장점 : 운용속도 우수, 선이 없이 소형의 독립적인 장치 설계 가능, 사전 계획 없이 통신 가능
  - 단점 : 유선에서 제공하는 수준보다 낮은 서비스 품질 및 속도

2. 무선 랜 네트워크 보안
- 위협 요소
  - 네트워크 도청 : 훔쳐보기, 데이터 수집
  - 내부망 침투 : 사내 정보 유출, 해킹 범죄의 근원지, 각종 바이러스 투입
  - 정보 외부 전송 : 불법 AP 사용, 물리적 사용

- 무선 랜 보안 요소
  - 인증<공유키 인증, 개방 시스템 인증>
  - 허가
  - 무결성
  - 기밀성

- 801.1x 프레임워크
  - 내부망 침투 >> 강력한 인증 >> IEEE 802.1x + 인증, 암호화
  - 네트워크 도청 >> 동적 암호화 >> IEEE 802.1x + 인증, 암호화
  - 정보 외부 전송 >> AP 불법 접근 금지 >> 관리 수칙 준수

- 위협요소 대응
  - 무선 랜 보안 규격 : IEEE 802.1x <호환> IEEE 802.11i
  - 인증 방식 : EAP(Extensible Authentication Protocol) - MD5, EAP-TLS(Transport Layer Security)
  - 암호화 : WEP, WPA / TKIP(Temporal Key Inteqrity Protocol), CCMP(Counter Mode CBC-MAC Protocol)

- 위협요소 대응 기술
  - WEP(Wired Equivalency Protocol) : 인증 + 암호화 제공, 확장 가능 인증 프로토콜(EAP)을 통해 사용자 인증  
(대칭키 암호 알고리즘) RC4 스트림암호 사용 네트워크 상의 모든 사용자와 모든 패킷에 '하나의 키'를 사용하므로 취약
  - WPA(WIFI Protected Access) : 인증 + 암호화 제공, 확장 가능 인증 프로토콜을 통해 사용자 인증  
임시키 무결성 프로토콜(TKIP)을 통해 데이터 암호화 >> 블록 암호 알고리즘인 AES를 사용하며 특정 시간 및 패킷 단위로 '키를 변경'  
하여 안전 CCMP로 발전

3. 무선 랜 네트워크 인증 및 암호
- EAP-RC4 인증(Challenge_Response)
단말기 >> 1. 연결요청 >> 무선공유기 >> 2. 임의의 숫자생성 3. 생성된 숫자 전송 >> 단말기 4. 수신 숫자를 KEY로 암호화하여 재 전송  
무선공유기 >> 5. 수신 숫자를 KEY로 복호화 6. 복호된 값을 처음 생성한 숫자와 비교 7. 동일한 경우 연결 허용 

- EAP-TLS(Transport Layer Security)
단말기 >> 1. 연결요청 >> 무선공유기 >> 2. 지원 알고리즘 교환 >> 단말기 >> 3. 단말기 인증서 전송 >> 무선 공유기 >> 4. 단말기 인증서 확인  
5. 서버 인증서 전송 >> 단말기 >> 6. 서버 인증서 확인 7. 확인되었음을 전송 >> 무선 공유기 >> 8. 연결허용 >> 단말기 >> 9. 인증 완료 

- RADIUS(Remote Authentication Dial-In User Service) 인증 : 중앙집중형 서버를 운영하여 향상된 보안 기능 제공
  - 사용자는 AP에 접속을 요청
  - AP는 무선 랜 네트워크에 사용자가 로그인할 때까지 접속 차단
  - RADIUS 서버는 사용자에 인증 Challenge를 전송
  - 사용자는 Challenge에 대한 응답으로, ID와 PASSWORD에 대한 해시 값을 구해 RADIUS 서버에 전송
  - RADIUS 서버가 사용자 관리 정보를 사용해 Challenge의 응답에 대한 해시 값을 구하여 사용자의 해시 값과 비교
  - RADIUS 서버가 사용자를 인증하면 이러한 과정을 역으로 수행하여 사용자가 RADIUS 서버를 인증하여 사용자와 상호 인증 수행
  - 사용자와 RADIUS 서버의 상호 인증이 완료되면, 네트워크 접근을 위한 사용자의 수준을 정의하고 사용자를 구별할 수 있는 WEP키를 결정
  - RADIUS 서버는 세션키라 하는 WEP 키를 유선 랜을 통해 AP로 전송
  - AP는 세션키로 브로드캐스트 키를 암호화하여 사용자에 암호화 키 전송
  - 사용자는 브로드캐스트 된 암호화된 키를 세션키를 사용 복호화하고 사용자와 AP는 WEP를 활성화하며, 세션 성립 시간 동안 통신에  
세션과 브로드캐스트 키를 사용

- TKIP 암호 (Temproal Key Integrity Protocol)
- CCMP 암호 (Counter Mode CBC-MAC Protocol)
